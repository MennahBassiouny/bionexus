Command line & samtools

The shell basics plus the genomics toolkit you use daily.

Navigating & files

pwdprint working directory (where am I)
ls -lhlist files, human-readable sizes
cd dir / cd ..change directory / go up one
mkdir -p a/bmake folders (and parents)
cp src dstcopy (use -r for folders)
mv src dstmove / rename
rm filedelete (no undo, careful)

Looking inside & filtering

cat / less fileprint / page through a file (q to quit)
head -n 20 / tailfirst / last lines
wc -l filecount lines
grep "pat" filefind lines matching a pattern
cmd | cmd2pipe output into the next command
cmd > out.txtredirect output to a file

samtools (BAM/SAM)

samtools view -b in.sam > out.bamSAM → BAM
samtools sort in.bam -o sorted.bamsort by position (needed before indexing)
samtools index sorted.bamcreate the .bai index
samtools flagstat in.bamquick mapping summary
samtools view in.bam chr1:1-1000reads in a region (needs index)
samtools depth in.bamper-base coverage
samtools faidx ref.faindex a FASTA reference

bcftools (variants)

bcftools mpileup -f ref.fa in.bampile up reads vs reference
... | bcftools call -mv -Oz -o out.vcf.gzcall variants (-m multiallelic, -v variants only, -Oz gzip)
bcftools view in.vcf.gzview / filter a VCF
bcftools stats in.vcf.gzsummary statistics
Order matters: sort → index before region queries. You cannot index an unsorted BAM.

File formats

What each bioinformatics file holds, at a glance.

FASTA — sequences

>seq_id description
ACGTACGTACGTACGT

> header line, then the sequence. DNA, RNA, or protein.

FASTQ — reads + quality

@read_id              # 1. name
ACGTACGT              # 2. sequence
+                     # 3. separator
IIIIFFFF              # 4. per-base quality
Q201 in 100 chance of error (99%)
Q301 in 1000 (99.9%) — common target
Q401 in 10,000 (99.99%)

GTF / GFF — gene annotation

Tab-delimited features (gene, exon, CDS) with coordinates on a reference. Maps positions to genes.

SAM / BAM — aligned reads

SAM is text; BAM is the compressed binary version. 11 mandatory columns:

QNAMEread name
FLAGbitwise info (mapped, paired, strand…)
RNAME / POSreference name & 1-based position
MAPQmapping quality
CIGARhow the read aligns (M/I/D/S…)
SEQ / QUALthe read bases & qualities

VCF — variants

CHROM / POSwhere the variant is
REF / ALTreference vs alternate allele
QUALconfidence the variant is real
FILTER / INFOpass/fail & annotations
FORMAT + samplesper-sample genotypes (e.g. 0/1)

BED — regions

chr1   200   1000   name   # 0-based start
BED is 0-based start; GTF/VCF are 1-based. A frequent off-by-one trap.

conda & Git

Environments that never break, and version control you can trust.

conda — environments

conda create -n myenv python=3.11new environment
conda activate myenvturn it on
conda install -c bioconda samtoolsinstall a tool from bioconda
conda listwhat's installed
conda deactivateturn it off
conda env export --from-history > environment.ymlsave a portable spec
conda env create -f environment.ymlrecreate it anywhere
One environment per project. Never install into base. Pin versions that matter.

Git — version control

git clone <url>copy a repo to your machine
git statuswhat changed
git add file / git add .stage changes
git commit -m "msg"save a snapshot
git push / git pullsend to / get from GitHub
git log --onelinehistory
git checkout -b featurenew branch
# .gitignore — never commit these
*.bam
data/
.env        # secrets / keys
The loop: edit → add → commit → push. Commit small and often.

Stats & DESeq2

The numbers you must interpret, and the differential-expression workflow.

Statistics quick reference

p-valueprobability of data this extreme if the null were true. Not the chance the result is real.
adjusted p / padj / FDRp-value corrected for testing thousands of genes (Benjamini-Hochberg). Filter on this, not raw p.
effect sizehow big the change is (e.g. log2 fold change). Always report alongside the p-value.
log2FC = 12× change; = 3 is 8×. Negative = down.
multiple testing20,000 tests at p<0.05 ≈ 1,000 false hits by chance. Always correct.
A tiny p-value with a near-zero effect size is statistically significant but biologically boring. Use both axes.

DESeq2 workflow (R)

# 1. load
library(DESeq2)

# 2. build dataset: counts + sample table
dds <- DESeqDataSetFromMatrix(
  countData = counts,
  colData   = coldata,
  design    = ~ condition)

# 3. run the analysis
dds <- DESeq(dds)

# 4. get results (treated vs control)
res <- results(dds,
  contrast = c("condition","treated","control"))

# 5. shrink fold changes for ranking/plots
res <- lfcShrink(dds, coef=2, type="apeglm")

# 6. significant genes
sig <- subset(res, padj < 0.05)
