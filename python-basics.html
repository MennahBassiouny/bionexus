Python Basics for Bioinformatics

The core Python you'll actually use - variables, lists, dictionaries, loops, functions, and reading files - taught with DNA and genes instead of abstract examples.

🟢 Beginner ⏱️ ~1.5 hr 🐍 Python

You've got Python installed (if not, do the install lesson first). Now let's learn just enough to be dangerous. Follow along in whichever tool you set up in the install lesson - VS Code, PyCharm, Jupyter, or Google Colab all work equally well here. If you're working in a terminal, activate your environment first (bionexus\Scripts\activate on Windows, source bionexus/bin/activate on Mac/Linux) and type python to start. Whichever you choose, type each example yourself - that's how it sticks.

1. Variables & strings

A variable is a name for a piece of data. In bioinformatics, a DNA sequence is just a string - text made of A, C, G, and T.

dna = "ATGCGTACGTTAG"
print(dna)
print(len(dna))        # length of the sequence → 13
print(dna.count("G"))   # how many G's → 3
print(dna[0])           # first base (counting starts at 0!) → A

Decode the jargon: zero-indexing

Python counts from 0, not 1. So dna[0] is the first base and dna[1] is the second. This trips up everyone at first - biologists especially, since we usually call the first base "position 1." Just remember: code starts at zero.

2. Doing real work with strings

# GC content: the fraction of bases that are G or C - a real, useful metric
gc = (dna.count("G") + dna.count("C")) / len(dna)
print(round(gc * 100, 1), "% GC")

# Replace lets us transcribe DNA → RNA (T becomes U)
rna = dna.replace("T", "U")
print(rna)
✍️ Try it: change the dna string and re-run. What GC content do you get for a sequence of all A's and T's?

3. Lists - ordered collections

genes = ["TP53", "BRCA1", "EGFR", "MYC"]
print(genes[1])          # BRCA1
genes.append("KRAS")     # add one to the end
print(len(genes))        # 5

4. Dictionaries - labelled lookups

A dictionary stores key → value pairs. Perfect for things like "how many of each base?"

counts = {"A": 10, "C": 7, "G": 9, "T": 12}
print(counts["G"])       # 9
counts["N"] = 2          # add an "unknown base" entry

5. Loops - do something to each item

for gene in genes:
    print(gene, "has", len(gene), "letters")

# Count each base in a sequence by looping
tally = {}
for base in dna:
    tally[base] = tally.get(base, 0) + 1
print(tally)

Why loops matter

A genome has millions of bases. You'll never process them by hand - you write the instruction once and the loop applies it to everything. This is the moment programming starts paying off.

6. Functions - reusable recipes

When you do something more than once, wrap it in a function so you can reuse it.

def gc_content(seq):
    """Return the GC percentage of a DNA sequence."""
    g = seq.count("G") + seq.count("C")
    return round(g / len(seq) * 100, 1)

print(gc_content("ATGCGCGC"))   # 75.0
print(gc_content("AAATTT"))     # 0.0

Your turn. Run this in your browser, then edit the sequences (or the function) and run again.

7. Reading a file

Real data lives in files. Here's the pattern for reading one line by line:

with open("sequence.txt") as f:
    for line in f:
        line = line.strip()      # remove the invisible newline
        print(line)

Decode the jargon: Biopython

For real sequence files (FASTA, FASTQ) you won't parse them by hand - a library called Biopython does it for you. You installed it in the previous lesson. You'll use it in the project next.

Check your understanding
