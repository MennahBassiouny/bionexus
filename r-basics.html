R Basics: Data Frames & Plotting

The R you'll use constantly — vectors, data frames, the handful of tidyverse verbs that do 80% of data work, and your first ggplot — built up gently and ready for real data.

🟢 Beginner ⏱️ ~1.5 hr 📊 R · tidyverse

Open RStudio and type each line into the Console. Better yet, open a new R Script (File → New File → R Script) and run lines with Ctrl/Cmd+Enter — that way you keep a record of what you did.

1. Vectors — R's basic building block

R loves vectors: an ordered set of values of the same type. Most R operations work on whole vectors at once, which is what makes it so good for data.

expression <- c(12.4, 8.1, 15.7, 3.3, 9.9)   # c() = "combine"
mean(expression)     # average → 9.88
max(expression)      # 15.7
expression * 2       # multiplies EVERY value at once
length(expression)   # 5

Decode the jargon: <-

In R, <- is the assignment arrow — it means "store this on the right into the name on the left." (= also works, but R people use the arrow. RStudio types it for you with Alt+−.)

2. Data frames — the spreadsheet of R

A data frame is a table: columns are variables, rows are observations. It's exactly how gene expression tables, sample sheets, and clinical data are shaped.

genes <- data.frame(
  gene  = c("TP53", "BRCA1", "EGFR", "MYC"),
  length_kb = c(19.1, 81.2, 188.3, 5.3),
  chromosome = c("17", "17", "7", "8")
)
genes            # print the whole table
head(genes)      # first rows (handy for big tables)
genes$length_kb  # pull out one column by name with $

3. The tidyverse verbs that do most of the work

Load the tidyverse, then meet five verbs that cover the vast majority of everyday data wrangling. The |> "pipe" passes the result of one step into the next — read it as "and then."

library(tidyverse)

genes |>
  filter(length_kb > 10) |>          # keep rows matching a condition
  arrange(desc(length_kb))           # sort, largest first

The five you'll reach for again and again:

  • filter() — keep rows that match a condition
  • select() — keep only certain columns
  • mutate() — add or change a column
  • summarise() — collapse to a summary (mean, count…)
  • group_by() — do the above per group
# Add a column, then average gene length per chromosome
genes |>
  mutate(length_bp = length_kb * 1000) |>
  group_by(chromosome) |>
  summarise(avg_length = mean(length_kb))
✍️ Try it: change filter(length_kb > 10) to > 50. Which genes survive?

4. Your first plot with ggplot2

ggplot2 (part of the tidyverse) builds plots in layers: start with the data, say what goes on each axis (aes), then add a geom — the kind of mark to draw.

ggplot(genes, aes(x = gene, y = length_kb)) +
  geom_col(fill = "#5b9cf8") +
  labs(title = "Gene lengths", y = "Length (kb)") +
  theme_minimal()

Why ggplot looks weird at first (the +)

You build a ggplot by adding layers with +: data, then axes, then geoms, then labels and themes. Once it clicks, it's a wonderfully logical way to compose any figure — and it produces publication-quality plots by default.

5. Reading your own data

Real data usually arrives as a CSV. Reading it is one line:

my_data <- read_csv("my_file.csv")
glimpse(my_data)   # a quick look at columns and types

Check your understanding

What does the pipe |> do?

It passes the result of the left-hand side into the first argument of the right-hand side, so you can chain steps in reading order: "take genes, and then filter, and then sort."

What's the difference between filter() and select()?

filter() keeps rows (observations) that match a condition; select() keeps columns (variables). Rows = filter, columns = select.

Why build plots with + layers?

Because it lets you compose any figure from simple, reusable pieces — swap a geom_col for a geom_point and you have a different chart of the same data, with everything else unchanged.
