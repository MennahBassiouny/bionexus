Installing Python

By the end of this lesson, Python will be running on your own computer — Windows, Mac, or Linux — and you'll have a clean workspace ready for real bioinformatics work.

🟢 Beginner ⏱️ ~30 min 🐍 Python 💻 Windows · macOS · Linux

What we're installing, and why

We'll install Python using Miniconda rather than the bare installer from python.org. Here's why that's the right call for a bioinformatician: Miniconda comes with conda, a tool that installs scientific packages cleanly and lets you keep separate "environments" so one project never breaks another. Almost every bioinformatics tutorial you'll meet assumes conda — so installing it now saves you pain later.

Decode the jargon: environment

An "environment" is a self-contained box holding a specific Python version and set of packages. Project A can use one box, Project B another, and they never collide. It's the single best habit for avoiding the dreaded "it worked yesterday" problem.

Step 1 — Download & install Miniconda

Go to the official download page: anaconda.com/download (or docs.conda.io) and grab the Miniconda installer for your operating system. Then follow the section for your OS below.

🪟 Windows Windows 10/11

  1. Download the Miniconda Windows 64-bit installer (.exe).
  2. Double-click it and follow the wizard. Accept the defaults — installing "Just Me" is fine.
  3. When it finishes, open Anaconda Prompt from the Start menu (search for it). This is your Python-aware terminal.

🍎 macOS Intel & Apple Silicon

  1. Pick the right installer: Apple Silicon (M1/M2/M3/M4) or Intel. Not sure? Click the Apple menu → About This Mac → look at "Chip".
  2. Download the .pkg graphical installer and run it, or use the command-line installer in Terminal:
# Example for Apple Silicon — adjust the filename to the one you downloaded
bash ~/Downloads/Miniconda3-latest-MacOSX-arm64.sh

Close and reopen Terminal when it's done.

🐧 Linux Ubuntu / Debian / Fedora …

  1. Download the Linux .sh installer, then run it from a terminal:
bash ~/Downloads/Miniconda3-latest-Linux-x86_64.sh

Press Enter / type yes through the prompts, then close and reopen the terminal.

Step 2 — Check it worked

In your terminal (Anaconda Prompt on Windows, Terminal on Mac/Linux), type:

python --version
conda --version

You should see a version number for each (for example Python 3.13.x). If you do — Python is installed. 🎉

Step 3 — Make your first environment

Let's create a dedicated workspace for your BioNexus learning, with the packages you'll use throughout:

# Create an environment called "bionexus" with some core tools
conda create -n bionexus python jupyter pandas numpy matplotlib biopython

# Switch into it (do this every time you work)
conda activate bionexus

When the environment is active, you'll see (bionexus) at the start of your prompt. That's how you know which box you're working in.

Prefer not to use conda?

You can install Python straight from python.org and use its built-in venv for environments and pip for packages. That works fine — but conda handles tricky scientific packages more smoothly, which is why we recommend it for bioinformatics.

✅ You're ready when…

python --version prints a version number, and your prompt shows (bionexus) after you activate the environment. That's everything you need for the next lesson.

Stuck? Common snags

"conda is not recognized" on Windows

You're probably in the regular Command Prompt. Close it and open Anaconda Prompt from the Start menu instead — that's the one that knows about conda.

"command not found: conda" on Mac/Linux

Close your terminal completely and open a fresh one. If it still fails, the installer may not have run its setup step — re-run the installer and answer yes when it asks to initialize conda.

I installed the wrong Mac version (Intel vs Apple Silicon)

No harm done. Delete the Miniconda folder in your home directory and reinstall with the correct one. Apple Silicon Macs can run the Intel build, but the native Apple Silicon build is faster.
